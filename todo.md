# XSOC 智能安全运营平台 项目规划
## 📋 项目概述
基于多智能体协作的安全智能运营层，**仅负责告警产生后的全流程处理**，无需关注上游安全检测设备和XDR/SOC的实现。
- 触发点：仅接收两种输入 → ① XDR中心推送的告警事件 ② 用户手动输入的告警信息
- 处理流程：事件研判 → 溯源分析 → 风险处置 → 数据归档，完全匹配blueprint中的智能运营层设计

## 🎯 核心特性
1. **半自动化运营**: 低风险事件全流程自动化处置，高风险事件引入HITL人工确认
2. **多智能体协作**: 四位专业安全专家智能体分工协作，各司其职
3. **双模式能力集成**: 本地API封装为Skills调用 + 云端能力通过MCP协议接入
4. **风险动态评估**: 处置前自动进行风险度评分(0-10分)，阈值可通过环境变量配置
5. **最小化影响**: 处置策略遵循不影响业务的最小化闭环原则
6. **可追溯性**: 完整事件链路记录，所有操作留痕，便于审计复盘
7. **多维度归档**: xDR中心结构化存储 + 钉钉AI表格可视化展示

- 数据序列化: Pydantic
- 日志: Structlog
- 配置管理: Pydantic Settings
- 异步框架: Asyncio/Aiohttp
- 测试框架: Pytest

## ⚠️ 注意事项
1. 所有API密钥/敏感信息通过环境变量或secrets配置文件管理，不硬编码
2. 所有Skill/MCP调用添加超时、重试、熔断机制，避免影响上游系统
3. 风险评分高于配置阈值时，强制触发HITL人工确认，无人工授权不得执行高风险处置
4. 处置操作遵循最小权限原则，只执行必要的处置动作，避免影响业务
5. 全程操作留痕，所有动作、决策依据、人工确认记录都要保存到事件链路中
6. API接口开发待用户提供详细接口文档后再进行实现
7. **项目边界明确**：仅实现blueprint中的「智能运营层」，不涉及上游安全检测组件、XDR/SOC层的开发，不做告警采集、预处理等功能
